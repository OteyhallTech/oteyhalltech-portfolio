Screenshots & Step-by-Step Walkthrough
Step 1
Bucket Creation
ActionCreated an S3 bucket named oteyhalltech-s3-lab-2026 — visible in the S3 bucket list confirming successful provisioning.
Why It MattersBuckets are the foundational storage containers in Amazon S3. Every object stored in S3 must live inside a bucket, and bucket names must be globally unique across all AWS accounts.
Skill DemonstratedCloud storage setup and resource creation in AWS — understanding bucket naming conventions and regional deployment.
Step 2
Creation Confirmed
ActionAWS Management Console displayed a success confirmation banner after the bucket was created, validating the provisioning completed without errors.
Why It MattersConfirming resource creation through console feedback is a critical habit in cloud operations — it ensures resources are provisioned as expected before proceeding.
Skill DemonstratedAWS Console navigation and cloud resource validation — interpreting success states and confirming operations in the AWS environment.
Step 3
Folder Organization
ActionCreated a logical folder named Project-Files inside the S3 bucket to organize uploaded objects into a structured hierarchy.
Why It MattersS3 uses a flat storage model, but prefixes (folders) simulate directory structures. Organizing objects this way improves scalability, access control granularity, and long-term manageability at scale.
Skill DemonstratedObject organization and data structuring in AWS — applying best practices for S3 folder hierarchies and prefix-based organization.
Step 4
File Upload & Verification
ActionUploaded a JPEG file into the Project-Files folder and confirmed it appears correctly inside the bucket, verifying object storage integrity.
Why It MattersUploading and verifying objects demonstrates the core S3 workflow — storing data in the cloud and confirming it is accessible. Verification is essential to ensure no corruption or misconfiguration occurred during upload.
Skill DemonstratedObject upload and validation in AWS — confirming data integrity and successful cloud storage operations through the S3 console.
Step 5
Bucket Properties Review
ActionReviewed the bucket's properties page, including region, ARN, versioning status, and server-side encryption settings in the AWS Management Console.
Why It MattersUnderstanding configuration settings is critical for security, compliance, cost optimization, and performance. The ARN is required for IAM policies; versioning protects against accidental deletion; region affects latency and data residency.
Skill DemonstratedCloud configuration analysis and resource inspection — reading and interpreting AWS resource metadata to support informed infrastructure decisions.
Step 6
Permissions & Access Control
ActionReviewed the bucket's permissions tab, including Block Public Access settings and ACL configuration to understand the access control posture of the bucket.
Why It MattersS3 bucket permissions are a leading cause of cloud security incidents. Reviewing and locking down public access by default is a security best practice aligned with AWS's Shared Responsibility Model and least-privilege principles.
Skill DemonstratedCloud security analysis and access control configuration — identifying and validating bucket-level permissions to prevent unintended data exposure.
